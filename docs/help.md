# PlcsimGateway - руководство пользователя

PlcsimGateway.Gui.exe соединяет Siemens PLCSIM с внешними S7-клиентами по
сети. HMI, SCADA, WinCC Runtime, EasyBuilder Online Simulation или физическая
панель подключаются к IP-адресу ПК-шлюза. Шлюз пересылает обмен в PLCSIM
через `S7ONLINE / PLCSIM.TCPIP.1`.

## Быстрый запуск

Пример:

- IP PLCSIM: `192.168.40.10`
- Network IP / IP ПК-шлюза: `192.168.40.50`
- порт: `102`

1. Запустите PLCSIM из проекта PLC.
2. В настройках подключения HMI / WinCC / SCADA укажите адрес PLC равным IP
   ПК-шлюза, например `192.168.40.50`.
3. Запустите `PlcsimGateway.Gui.exe` от имени администратора.
4. Укажите **PLCSIM IP** = `192.168.40.10`.
5. Укажите **Network IP** = `192.168.40.50`.
6. Выберите `standard (port 102)`.
7. Нажмите **Save IPs**.
8. Нажмите **Start**.
9. В Siemens **Set PG/PC Interface** установите
   `S7ONLINE -> PLCSIM.TCPIP.1`. Если выше `S7ONLINE` отображаются другие
   подключения, установите их в `None`.
10. Запустите HMI Runtime, панель, SCADA Runtime или другой S7-клиент. В шлюзе
    должна появиться активная сессия, а счетчики должны начать расти.

## Профили

| Профиль | Описание |
|---------|----------|
| `standard (port 102)` | Основной режим. Принимает подключения на сетевом IP этого компьютера по TCP/102. Подходит для внешних HMI-панелей, WinCC Runtime, EasyBuilder, других ПК и локальной online-симуляции. |
| `loopback fallback (port 1102)` | Резервный локальный режим. Слушает `127.0.0.1:1102`. Используется только для диагностики на этом же ПК, если клиент умеет работать с нестандартным портом. |

## Поля главного окна

| Поле | Назначение |
|------|------------|
| **Network IP** | IP-адрес сетевого адаптера, на котором шлюз принимает подключения. Если IP компьютера изменился, обновите это поле, нажмите **Save IPs** и перезапустите профиль. |
| **PLCSIM** | Адрес CPU в симуляторе и номер rack/slot, куда пересылаются запросы. Для текущего проекта используется Siemens S7-1200 CPU 1215C (TIA Portal V16). |
| **PID** | Идентификатор процесса Windows, внутри которого работает шлюз. |
| **Sessions** | Количество активных TCP/S7-клиентов, подключённых к шлюзу в данный момент. |
| **Health** | Краткое состояние цепочки `клиент - шлюз - PLCSIM`. |
| **Runtime** | Время последнего снимка диагностики в формате JSON. |

## Кнопки

| Кнопка | Действие |
|--------|----------|
| **Start** | Запускает выбранный профиль внутри процесса GUI. |
| **Stop** | Останавливает текущий шлюз. |
| **Restart** | Кратковременно разрывает все активные сессии и заново поднимает тот же профиль. |
| **Refresh** | Перечитывает состояние, логи и runtime-диагностику. |
| **Save IPs** | Сохраняет текущие значения **Network IP** и **PLCSIM IP** в файл конфигурации. Уже запущенный профиль применит новые адреса только после **Restart** или следующего **Start**. |

## Диагностические счётчики и индикаторы

- **Active sessions** — список открытых клиентских TCP/S7‑сессий.
- **Last disconnects** — последние закрытые сессии с итоговыми счётчиками переданных/принятых данных.
- **Client PDU** и **Client bytes** — увеличиваются, когда клиент отправляет запросы в сторону шлюза/PLCSIM.
- **PLCSIM PDU** и **PLCSIM bytes** — увеличиваются, когда PLCSIM отвечает и шлюз возвращает ответ клиенту.

> **Диагностическая подсказка:**  
> Если клиентские счётчики растут, а счётчики PLCSIM — нет, это означает, что клиент отправляет запросы, но ответ от PLCSIM не приходит либо соединение с симулятором ещё не установилось.

## Если порт TCP/102 уже занят

При занятом порте `Network IP:102` кнопка **Start** вызовет двуязычный диалог для освобождения порта.  
Чаще всего порт 102 занят службой Siemens. GUI запрашивает права администратора и может:

- временно остановить процесс, занимающий порт,
- зарезервировать порт для шлюза,
- восстановить исходного владельца порта,
- после этого запустить шлюз.

## Примечания

Приложение вдохновлено NetToPLCsim и сделано для более понятного повторяемого
запуска PLCSIM-to-HMI стенда. Проверенные связки: TIA Portal V16 PLC + WinCC HMI
и TIA Portal V16 PLC + EasyBuilder HMI.
