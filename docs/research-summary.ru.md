# Краткий итог исследования

PlcsimGateway создан, чтобы сделать Siemens PLCSIM доступным для внешних
S7-клиентов по сети без запуска оригинального GUI NetToPLCsim.

Практическая цель - доступ HMI / SCADA к тегам, а не полноценная эмуляция PLC.

## Что подтверждено

- Внешние клиенты подключаются к ПК-шлюзу через ISO-on-TCP / TCP порт 102.
- Шлюз снимает транспортную обертку и пересылает полезную нагрузку S7 в PLCSIM через
  Siemens S7ONLINE.
- Трафик TIA Portal V16 / S7-1200 / WinCC Runtime наблюдался как S7commPlus.
- Для проверенных HMI-сценариев не потребовался полный разбор S7commPlus;
  достаточно было передавать полезную нагрузку через S7ONLINE.

## Проверенные клиентские сценарии

- WinCC Runtime на втором ПК.
- WinCC Runtime на том же ПК, где запущен PLCSIM.
- Несколько WinCC Runtime клиентов одновременно.
- EasyBuilder Online Simulation на том же ПК, где запущен PLCSIM.
- EasyBuilder Online Simulation с другого ПК.
- Физическая HMI-панель Weintek.

## Что не покрыто

- Полная эмуляция PLC без PLCSIM.
- PROFINET discovery / Accessible Nodes.
- Инженерные сценарии шире доступа HMI / SCADA клиентов.
- Длительная проверка на множестве версий TIA Portal и сторонних SCADA
  Runtime.

Рассматривайте это как лабораторный инструмент и проверяйте свою конфигурацию
перед использованием в рабочем процессе.
