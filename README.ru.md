# PlcsimGateway

[English version](README.md)

PlcsimGateway - это Windows-приложение-шлюз, которое позволяет внешним
S7-клиентам подключаться к Siemens PLCSIM CPU по сети.

Типовая цепочка:

```text
WinCC / HMI / SCADA / S7-клиент
  -> IP ПК-шлюза:102
  -> PlcsimGateway.Gui.exe
  -> S7ONLINE / PLCSIM.TCPIP.1
  -> Siemens PLCSIM CPU
```

Приложение разработано после практической работы с NetToPLCsim. NetToPLCsim
показал, что такой сценарий возможен, но в реальном стенде запуск и диагностика
часто оказывались неудобными и плохо повторяемыми. PlcsimGateway сохраняет ту
же практическую цель, но делает запуск более понятным: IP-адреса редактируются
в окне, видны активные сессии, есть счетчики обмена и меньше ручных действий.

Инструмент опубликован в открытый доступ для лабораторных стендов, проверки
HMI, SCADA и интеграционных сценариев. Тесты на других системах, исправления,
доработки и любая обратная связь приветствуются.

## Проверенные сценарии

Разработка и тестирование выполнялись на:

- TIA Portal V16, проект PLC + WinCC HMI;
- TIA Portal V16, проект PLC + EasyBuilder HMI.

Подтвержденные связки:

- PLCSIM на одном ПК, WinCC Runtime на другом ПК;
- PLCSIM и WinCC Runtime на одном ПК, при необходимости одновременно с WinCC
  Runtime на других ПК;
- PLCSIM и EasyBuilder Online Simulation на одном ПК;
- PLCSIM и EasyBuilder Online Simulation на одном ПК, при необходимости
  одновременно с EasyBuilder-симуляцией на другом ПК;
- PLCSIM на одном ПК и физическая HMI-панель Weintek через шлюз, при этом
  дополнительно могут подключаться симуляторы с других ПК.

На других версиях TIA Portal, других семействах PLC, других HMI Runtime и
других SCADA-системах приложение пока не проверялось, потому что под рукой не
было подходящих проектов и инструментов. Если клиент использует тот же S7-путь
через ISO-on-TCP / TCP порт 102, шлюз может сработать, но это нужно
подтвердить отдельным тестом.

## Быстрый старт

Пример адресов:

- IP PLCSIM CPU в проекте PLC: `192.168.40.10`
- сетевой IP ПК, где запущены PLCSIM и шлюз: `192.168.40.50`
- стандартный S7-порт: `102`

Шаги:

1. Запустите PLCSIM из проекта PLC. Например, симулируемый контроллер имеет IP
   `192.168.40.10`.
2. В настройках подключения HMI / WinCC / SCADA укажите адрес PLC равным IP
   ПК, на котором запущены PLCSIM и PlcsimGateway. В этом примере используйте
   `192.168.40.50`, а не `192.168.40.10`.
3. Запустите `PlcsimGateway.Gui.exe` от имени администратора.
4. В PlcsimGateway укажите:
   - `PLCSIM IP` = `192.168.40.10`
   - `NETWORK IP` = `192.168.40.50`
5. Выберите режим `standard (port 102)`.
6. Нажмите `Save IPs`.
7. Нажмите `Start`.
8. В Siemens `Set PG/PC Interface` убедитесь, что:
   - `S7ONLINE` указывает на `PLCSIM.TCPIP.1`;
   - все подключения выше `S7ONLINE`, если они отображаются в вашей системе,
     установлены в `None`.
9. Запустите HMI-симуляцию, HMI-панель, WinCC Runtime, SCADA Runtime или другой
   S7-клиент.
10. В PlcsimGateway должна появиться `ActiveSession`, а в логах и счетчиках
    должен начаться рост показателей обмена.

## Требования

- Windows.
- Установленный и рабочий Siemens PLCSIM.
- Siemens S7ONLINE / `S7onlinx.dll` на том же ПК.
- Настройка `S7ONLINE -> PLCSIM.TCPIP.1` в Siemens PG/PC Interface.
- .NET Framework 4.8.
- Сборка x86.
- Права администратора для открытия TCP/102 или обработки конфликта со службой
  Siemens, которая уже заняла порт.

В репозиторий не входят Siemens-бинарники, Siemens-документация, клиентские
проекты, HMI-проекты и сетевые захваты.

## Конфигурация

Основной профиль - `standard (port 102)`.

`NETWORK IP` - это IP-адрес ПК, на который подключаются внешние клиенты.
`PLCSIM IP` - это адрес симулируемого CPU из PLC-проекта.

Стартовые профили лежат в:

```text
config\gateway-profiles.json
```

Если распространять только `PlcsimGateway.Gui.exe`, GUI сможет создать
`config\gateway-profiles.json` рядом с exe при первом запуске.

## Диагностика

- Не появилась активная сессия: проверьте, что HMI смотрит на IP ПК-шлюза, а
  не напрямую на IP PLCSIM CPU.
- Сессия появилась, но счетчики не растут: проверьте, что HMI Runtime реально
  запущен и опрашивает теги.
- Растут клиентские счетчики, но не растут счетчики PLCSIM: проверьте, что
  PLCSIM запущен, проект загружен, а `S7ONLINE -> PLCSIM.TCPIP.1` выбран.
- TCP/102 занят: запустите GUI от имени администратора и разрешите ему
  обработать конфликт порта либо остановите конфликтующую службу Siemens
  вручную.
- Внешнее устройство не подключается: проверьте Windows Firewall и выбранный
  IP сетевого адаптера.

## Область применения и ограничения

PlcsimGateway - это шлюз к Siemens PLCSIM. Это не самостоятельный эмулятор
PLC, не замена финальным испытаниям на реальном оборудовании и не полноценная
реализация PROFINET discovery / Accessible Nodes.

Используйте его для лабораторных, симуляционных, HMI, SCADA и сценариев
разработки, где эти ограничения приемлемы.
